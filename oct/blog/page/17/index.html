Mysql修改用户密码.

Apr 3rd, 2011

今天配置Fedora的LAMP环境.没想到Fedora自己已经安装好了Mysql.但是他没有给ROOT设置密码.所以我们需要果断更改之. 首先.我们可以使用一些简单的工具.比如说phpmyadmin.默认情况下Fedora安装好的Mysql是ROOT无密码.而phpmyadmin是通过HTTP认证的.所以我们可修改/etc/phpMyadmin/config.inc.php的认证方式为config.然后把账号输入进去就行了(默认密码为空.那么留空就可以了.). 第二的话.我们可以通过.

UPDATE user SET password=PASSWORD("new password") WHERE user='name'; 
FLUSH PRIVILEGES;

第三种的话.我们可以通过.

SET PASSWORD FOR name=PASSWORD('new password');

最后一种.我们可以通过.

mysqladmin -u root "old password" "new password"

Perl_Exporter.pm使用.

Apr 2nd, 2011

一般情况下.我们大多使用Perl自带的库模块.直接use或者use qw.但是具体的实现都不是很了解.其实.我们不仅能控制导入什么.还能控制导出什么.控制导入的当然就是qw.控制导出就是通过Exporter模块的作用. Exporter模块有四个重要的变量.@ISA.@EXPORT.@EXPORT_OK.%EXPORT_TAGS. 下面直接看例子




Class.pm



!/usr/bin/perl



package Class;



use warnings;
use strict;



use Exporter;
BEGIN{



our @ISA = qw(Exporter);
our @EXPORT = qw(func1);
our @EXPORT_OK = qw(func2);
our %EXPORT_TAGS = ( 
        all =>  [qw(func1 func2)],
);  




}



sub func1{



print "Hello func1\n";




}



sub func2{



print "Hello func2\n";




}



1;


#!/usr/bin/perl
# use Class;
# use Class qw(func1 func2);
use Class qw(:all);
func1;
func2;

接下来解释下重点. @ISA指继承类.在类的继承那里用途比较大.这里记住继承Exporter就可以了. @EXPORT.就是在不用qw指定导入的时候默认导出的数据. @EXPORT_OK.就是在用qw指定的时候可以导出哪些内容. %EXPORT_TAGS.就是通过定义标签来导入一系列内容. 还有一个比较常用.@EXPORT_FAIL.这个就是指定不能导入的内容.具体的查看CPAN.

Apache_CGI配置.

Apr 1st, 2011

昨晚看了下Perl Web编程.CGI也应该适当的学习下.知识.越多越好…. 首先需要配置Apache来运行CGI程序.

cd /etc/apache2/site-avalible
vi default

默认的CGI路径为

ScriptAlias /cgi-bin/ /usr/lib/cgi-bin/

我们最好改变其路径为/var/www下.

mkdir /var/www/cgi-bin -p

然后修改default文件.

ScriptAlias /cgi-bin/ /var/www/cgi-bin/
AddHandler cgi-script cgi pl

需要注意的是.最好加上AddHandler不然会有很大的可能不能运行….不能运行事小.暴露了代码就不好了. ·

Samba_服务搭建

Apr 1st, 2011

Samba是一个网络服务器.用于linux/windows之间共享文件.如果只是进行linux之间的文件共享的话,应该选择更好的NFS.Samba常用于共享文件和共享打印机. Samba有两个服务器,一个是smb,一个是nmb.smb是主要启动服务器.让其他机器知道服务器共享了哪些目录.nmb如果不开启的话.其他用户只能通过IP进行访问. 启动和关闭Samba的时候只要对smb进行操作就可以了.

# Ubuntu
service smbd status/start/stop/restart
# Or
/etc/init.d/smb status/start/stop/restart
# 我的ubuntu中不带如上启动脚本.

查看Smaba服务器端口以及防火墙状态.确保Samba正常使用.

netstat -lnp | grep smbd
# iptables
service iptables status

先不去管配置细节.假设现在已经有一台配置好的Samba服务器.那么我们可以查看服务器上的资源.

smbclient -L //IP [-U user]

如果服务器设置为share模式.连接时仍然会提醒输入密码.此时直接回车就可以了. 使用smbclient登录服务器后.就可以进行一些命令行下的操作了.当然你也可以使用IDE的sambaclient.

# smbclient login success
?/help #HELP
! shellcommand #EXEC SHELLCOMMAND
cd directory #change work directory SERVER
lcd directory #change work directory CLIENT
ls/dir #list files
get file1 [file2] #download file1 and save as file2
mget file1 file2 file3 #download files from server
put file1 [file2] #upload file1 to server and save as file2
mput file1 file2 file3 #upload files to server
md/mkdir #mkdir directory
rd/rmdir #rmdir directory

挂在smb.可以使用mount也可以使用smbmount.推荐使用smbmount.因为mount需要设置挂在文件系统以及登录.比较复杂.当然IDE下操作也是相当不错的.

smbmount //IP/directory /mnt -o iocharset=utf8

Perl添加[模块]引用地址

Mar 30th, 2011

经常写Perl的会发现总会被提醒这样那样的@INC错误.其实就是包含路径错误.一般情况下如果做了正确的处理是不该出现这样的问题的.所以我还是把正确的处理方式说下.以免以后继续悲剧了. 首先.我们可以通过unshift把地址添加到@INC中.

BEGIN{
     unshift @INC,"/usr/share/perl";
}

或者就是.

use lib "/usr/share/perl";
#添加多行地址.
use lib qw(
      /usr/share/perl
      /usr/perl
);

Perl_List::Util模块使用.

Mar 29th, 2011

之前一直记着Perl是自带reverse,shuffle的.今天使用的时候发现shuffle不能使用.纠结了半天.上cpan一查.发现是List::Util模块的.这个不查不要紧.一查发现这个模块很给力. 果断安装(我不记得自己带不带了.我果断先安装了.).

perl -MCPAN -e shell
# shell提醒CPAN版本太旧了.
install CPAN
reload CPAN
install List::Util

安装好之后就上CPAN查看了下这个模块的具体使用方法. 首先是first的使用.first类似于grep/不过他只返回出现的第一个.

use List::Util qw(first);
$first = first{$_>13} @array;

然后他还有max.maxstr.min.minstr.shuffle.sum等函数.
