Perl POE框架 Tk编程

Apr 14th, 2011

前天晚上纠结于Perl Tk的编程.按常理来说.界面与监听daemon应该是线程关系.可是在Perl Tk中不好使啊.MainLoop了就不能操作任何其他的了.[不清楚具体原因.感觉上是可行的.但是就是搞不出来.google半天.也都说perl通过线程无法实现以上功能.所以猜测perl线程的实现可能有些不同.不是都是 POSIX吗?纠结…] POE框架是实现Perl/tk编程的一个有效方法.而且POE的应用远不仅仅限于此.所以这个框架还是需要好好研究的.我只记得网上有人说是讲单向过程虚拟成并行运算.可能有点神.等我研究好了再弄吧. 首先是官网上的一个例子.我适当加了注释.

#!/usr/bin/perl
use warnings;
use strict;

use Tk;
# Loop::TkActiveState 注册POE中的相关Tk全局变量
use POE qw( Loop::TkActiveState );
# 初始化响应机制
POE::Session->create(
  inline_states => {
    _start   => \&ui_start,  #默认执行._stop默认终止执行.
    ev_count => \&ui_count,
    ev_clear => \&ui_clear,
  }
);

$poe_kernel->run();
exit 0;

# 补全机制内的响应函数
sub ui_start {
  my ($kernel, $session, $heap) = @_[KERNEL, SESSION, HEAP];

  $poe_main_window->Label(-text => "Counter")->pack;
  # %$heap内为全局变量.
  $heap->{counter_widget} =
    $poe_main_window->Label(-textvariable => \$heap->{counter})->pack;
  $poe_main_window->Button(
    -text    => "Clear",
    -command => $session->postback("ev_clear")
  )->pack;
  # 转入下一响应
  $kernel->yield("ev_count");
}

sub ui_count {
  $_[HEAP]->{counter}++;
  $poe_main_window->update;    # Needed on SunOS & MacOS-X
  $_[KERNEL]->yield("ev_count");
}

sub ui_clear {
  $_[HEAP]->{counter} = 0;
}

需要注意的是:

perl -MCPAN -e shell
install POE
# 还需要安装POE::Loop::TkActiveState才能正常使用以上代码.
install POE::Loop::TkActiveState

接下来重点解释下. POE中允许定义一堆事件或者可以叫做响应机制的东西.然后默认由start开始执行.我们可以在过程中通过用户的操作或者某种判断yield到另一个事件中.通过POE这个框架.我们不需要关注界面与业务逻辑的线程关系.至于POE的其他应用等待我研究. POE中的Tk主窗口变量已经定义.为$poe_main_window.然后就是我们可以定义Tk组件然后保存导全局变量$[HEAP]中.然后就可以在其他事件中使用. 还有就是POE的tk中的button的-command响应通过$_[SESSION]->postback来进入到响应事件中. 大概就这么多.`

Perl Select Socket实现

Apr 12th, 2011 

use strict;
use IO::Socket;
use IO::Select;

$|++;  #open autoflush
my $s = new IO::Socket::INET(
         LocalAddr   =>   'localhost',
         LocalPort    =>   1234,
         Listen         =>    10,
         Proto          =>    'tcp',
) or die $@;
# 创建一个select 目标
my $read_set = new IO::Select(); 
$read_set->add($s);

while(1){
    #IO::Select的select方法.第一个参数设置为select目标.
    #然后方法会一直检查可读目标.直到有可用句柄返回.
    #返回一个列表.第一个参数表示可读的句柄数组引用.
    my ($rh_set) = IO::Select->select($read_set, undef, undef);
    foreach my $rh (@$rh_set){
          # 如果当前可读的socket等于侦听socket.那么说明有请求进来.
          # 应该及时accept.把已经建立的socket句柄加入检查队列.
          if($rh == $s){
                my $ns = $rh->accept();
                $read_set->add($ns);
          }else{
          # 使用sysread读取数据.每次读取32字节.并且只处理这32字节.
          # 如果客户发送多于32字节.会分为几次进行处理.
          # 如果多个客户端同时发送.那么处理是无序的.
          # 避免使用<>方法来读取socket.
          # 因为<>是面向行的.perl IO对它进行了缓冲.IO::Select看不到这个缓冲.
                my $buf = undef;
                if(sysread($rh, $buf, 32)){
                       print $buf;
                }else{
                       #没有更多的数据.在队列中除去
                       $read_set->remove($rh);
                }
         }
 }

use strict;
use IO::Socket;

my @childs;
# fork 3个进程同时发送数据
for( 1..3){
     my $child = fork;
     if($child){
           push(@childs, $child);
     }else{
           # 创建到server的连接
           my $sock = new IO::Socket::INET(
                    PeerAddr =>  'localhost',
                    PeerPort  =>  1234,
                    Proto       =>   'tcp',
           ) or die $@;
           for (1..10){
                    print $sock random() . "\n";
                    select(undef, undef, undef, 0.25);
                    # 每发送一次.就休眠0.25秒.
           }
           $sock->close or die $@;
           exit 0;
      }
for (@childs){
      my $tmp = waitpid($_, 0);

}

Perl 多线程Socket实现.

Apr 12th, 2011 

#!/usr/bin/perl -w
use strict;
use posix;
use IO::Socket;
$| = 1;
my $main_sock = new  IO::Socket::INET(
        LocalHost   =>  '172.29.143.87',
        LocalPort   =>  9999,
        Listen      =>  10, 
        Proto       =>  'tcp',
        Reuse       =>  1,  
) or die "Socket couldn't create:$!\n";

while(my $new_sock = $main_sock->accept()){
    my $pid = fork;
    if($pid == 0){ 
        while(defined(my $line = <$new_sock>)){
            if($line =~ /(bye|quit)/i){
                print "Disconnect\n" and last;
            }   
            print $line;
            print $new_sock "Got It";
        }   
        exit(0);
    }   
}
close($main_sock);

Iptables使用

Apr 7th, 2011

今天在宿舍连接实验室的机子.悲剧了. 连接不上.当时那个尴尬啊.[能够PING通.但是SSH和FTP都不能用].要不是因为下午有课.我就直接飞奔到实验室大干一场了.

service sshd status
service vsftpd status
chkconfig | grep -E 'sshd|vsftpd'

检查一切正常.我就无语了.然后我就在本地进行测试. 终端下ssh和ftp一切正常.然后我到浏览器中访问ftp.发现fedora提示selinux警告信息.当时我就明白了.防火墙没有关…. 之前用的发行版貌似默认都没有开启这些.然后我就仔细排查了下.在X下进行了相关改动.毕竟终端下的配置还不是很了解.这不看不要紧. 一看就发现fedora默认开启selinux和iptables. selinux貌似就是种安全策略吧.所以先不管它.先学习下iptables. iptables是一种在内核层面上的防火墙.可以通过检查数据包包头来进行拦截等.iptables默认有三个表.默认是filter表.我们针对这个表进行操作就可以了. 接下来直接说iptables的语法吧.

iptables [-t table] COMMAND MATCH TARGET

一般情况下我们操作的都是filter表.所以-t选项基本不怎么用了. 然后就是COMMAND部分.一般常用

-L #列出表内filter
-I  #插入到指定位置.
-A #插入Append
-D #删除指定
-F #全部删除

filter表下默认有三条链路.INPUT.OUTPUT.FORWARD.我们需要在COMMAND中指定操作链路.例如:

iptables -I INPUT 1 -s 172.29.143.87 -j REJECT

COMMAND后面就是MATCH条件了.一般常用的就是-s[来源地址]-d[目的地址].–sport[来源端口].–dport[目的端口].-p[协议]. 最后就是TARGET.需要由-j 进行引导.一般常用的有

ACCEPT
DROP
REJECT
RETURN

DROP的话不进行数据包的处理.REJECT返回拒绝信息.所以具体使用取决实际情况.建议DROP`

Linux Ps,top,kill,pkill,killall

Apr 6th, 2011

晚上没事干的时候大概学习了下mc的使用.感觉这个东西还是很方便的.虽然自己还不是很熟. mc可以通过Co来显示或者隐藏到后台.这样的应用我们可以把他作为一个后台程序一直进行着.非常方便.但是资源占用是我好奇的.

ps -ef | grep -v grep | grep -i mc
#
#wxm       7649  6381  0 19:15 pts/2    
#00:00:00 /usr/bin/mc -P /tmp/mc-wxm/mc.pwd.6381
#

显示的内容并没有资源消耗等信息.只是让你方便的得到了进程的PID.PPID等信息.这些并不是我想要的.所以我google了下.

ps aux | grep -v grep | grep -i mc
#wxm       7649  0.0  0.1  10160  2912 pts/2    S+   19:15
#0:00 /usr/bin/mc -P /tmp/mc-wxm/mc.pwd.6381

输出如上.这个可以很方便的知道CPU使用率.内存使用率.内存占用.硬盘占用等信息. 所以我们可以得到如此两个结论:在配合kill,pkill,killall等命令时候可以使用ps -ef.但是为了读取资源消耗信息的时候我们应该通过ps aux来获得. 还有一个命令是top.这个可以动态检测当前进程运行状态.也可以对进程进行一些基本的操作.这里只是强调下.top默认是5秒刷新一次.我们可以top -d 1来使top每秒刷新一次.还有就是top进入后.应该看下是否有僵尸进程… kill可以杀死进程.可以配合SIGNAL来进行杀除.killall是用来删除进程组.pkill是用来杀死程序.不过配合的是程序名.而不是pid.

pgrep -l mc
pkill mc

pkill -uroot

一次性杀死用户所有进程`
