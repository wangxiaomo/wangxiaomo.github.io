Zend_Json 学习

Mar 7th, 2011

Json是基于JavaScript一个子集的一种轻量级的数据交换形式.这种格式既容易阅读和编写,同时也易于计算机解析和生成.JSON采用完全独立于语言的文本格式,但是也使用了类似于C语言的语法. JSON语法: .名称/值 .{} 用于实现对象的包含 ., 用于分隔对象的不同属性,或者数组的元素. .[] 用于存放数组. *.: 用于表示键值的分隔符. 下面是一个简单的JSON对象实例:

{
    "addressbook":{
        "name":"Mary",
        "address":{
            "city":"New York",
            "zip":91919
        },
        "PhoneNumber":[
            1999,
            2000
        ]
    }
}

与同信息量的XML比起来.提高了解析效率.减轻了网络压力. 在Zend_Json组件中,可以通过Zend_Json::encode()方法将普通的PHP内容,如数组\对象或者其他类型的数据转化为JSON格式.

PHP中也有json操作的函数.其实Zend_Json可调用了它. 区别就是. Zend_Json可判断系统是否支持json系函数.存在则调用库函数.不存在.则调用Zend函数.

ZFW MVC入门例子

Mar 7th, 2011

Zend_Controller是Zend Framework的MVC体系的核心部分.MVC是一个用于分离应用逻辑和表现逻辑的设计模式.在该种模式下.所有的请求都通过前端控制器并分发到不同的控制器来处理.分发的过程基于请求的URL. Zend_Controller体系具有可扩展性.可以通过继承已有的类或者自定义新的类来实现各种接口和抽象类.也可以编写插件或者助手类来增强系统的功能. 使用Zend_Controller进行MVC编程.需要使用特定的目录结构.典型的目录结构如下:

---application
       controllers
             IndexController.php
       scripts
       helpers
       filters
       models
       views
             scripts
                 index
                     index.phtml
---html
       index.php
       .htaccess

其中.htaccess文件用来实现URL的重置.即当用户访问某资源时.会将其重新定位到指定的文件..htaccess的文件的代码如下:

RewriteEngine on
RewriteRule !\.(js|ico|gif|jpg|png|css)$ index.php

其中.行1表示重置引擎打开.行2表示除了访问js.ico.gif.ipg.png.css以外的文件时,都会被重置到index.php文件. 注意.htaccess用于Apache服务器.而且mod_rewrite需要打开.用于nginx服务器.需要在站点配置文件中include htaccess文件.

ZFW基础模块浏览

Mar 7th, 2011

框架是一种相对固定的设计模式.一个好的框架可以让开发者专注于商务逻辑的实现,并且把整个系统分成若干相互独立的层次,减少构件的耦合性.所以在实际开发中,能极大地提升开发效率. ZFW由若干部分组成.每个部分称为一个组件,一个组件实现特定的功能. .MVC组件 .核心组件 .数据库操作组件 数据操作.全文搜索引擎.PDF操作 .服务类组件 实现各类服务.RSS.XML-RPC.REST等 .国际化组件 具体情况如下: .Zend_Controller MVC的控制器. .Zend_View MVC的视图部分 .Zend_Json 此模块用于把PHP对象转换成Javascript对象符号.或者进行反向转换. .Zend_Loader 此模块用于实现对类的加载.使用此模块可以轻松的对ZFW其他组件进行加载. .Zend_Registry 此模块可以作为值与对象的存储容器. .Zend_Config 此模块可以从ini或者xml配置文件中读取数据. .Zend_Console_Getopt 此模块可以与命令和配置参数协同工作. .Zend_Filter 此模块提供对字符串进行过滤的方法.如isEmail方法和getAlpha方法等. .Zend_Validate 此模块实现校验器的功能. .Zend_Debug 此模块实现代码的调用处理. .Zend_Exception 此模块用于实现代码的异常处理. .Zend_Uri 此模块用于对URI进行操作. .Zend_Catch 此模块用于实现缓存的功能. .Zend_Session 此模块用于操作Session的数据. .Zend_Acl 此模块用于实现访问控制功能. .Zend_Auth 此模块用于实现认证适配器功能. .Zend_Memory 此功能用于实现在限制内存环境下的功能. .Zend_Mail 此模块使开发者能发送文本文件和多部分MIME格式电子邮件. .Zend_Mime 此模块用于为Zend_Mail组件解码MIME消息. .Zend_Log 此模块用于提供通用日志功能. .Zend_Version 此模块用于获取ZFW的版本.并进行版本比较. .Zend_Search_Lucene 此模块用于构建基于全文的搜索引擎. .Zend_Db 此模块用于访问数据库. .Zend_Pdf 此模块用于创建新的PDF.以及加载和编辑现有文档. .Zend_Feed 此模块用于RSS和Atom服务. .Zend_Server 此模块是其他服务类的基类.为其他Server类提供底层支持. .Zend_Http 此模块能使开发者便捷执行HTTP请求. .Zend_XmlRpc 此模块用于实现XmlRpc功能. .Zend_Rest 此模块用于实现REST服务. .Zend_Service 此模块可以使用网络上知名服务提供商的API. .Zend_Gdata 此模块可以使用Google提供的各类服务的API. .Zend_Locale 此模块为其他国际类的基类,为其他国际化类提供支持. .Zend_Translate 此模块为翻译组件.可以通过适配器实现不通语言的互译. .Zend_Date 此模块用于实现不同地域的日期时间功能. .Zend_Measure 此模块用于实现不通地域关于计量单位的转换.
