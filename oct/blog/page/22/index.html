Smarty缓存.

Mar 5th, 2011

缓存是smarty的一大特色.使用smarty模板的优势在于速度.速度是因为编译和缓存决定的.所以缓存需要正确设置. $caching变量.当该变量的值为true或者1时.缓存被开启.当使用display加载模板文件时.smarty会现在$cache_dir下找缓存文件.当缓存文件存在时.输出缓存文件.不存在时.重新编译文件. 可以使用$cache_lifetime设置缓存生存时期.默认是3600秒.可以将$caching设置成2.然后再设置$cache_lifetime设置缓存生存时间. 可以通过clear_cache和clear_all_cache来清除缓存.

clear_all_cache();
$smarty->clear_all_cache('3600');
$smarty->clear('index.tpl');
?>

问题是缓存生存时间从什么时候开始计算？以在服务器上编译时间为准？

Smarty内置函数.

Mar 5th, 2011

*.config_load

{config_load file=confname [section=section scope=local|parent|global]}

scope指定配置文件的作用域.local是本模板.parent是当前模板和父模板.global是所有模板. section可以指定读取的段.


.foreah .section 这两个函数查看之前写的文章或者官方文档.


*.include和insert include标签用来载入其他模板.

{include file=filename [assign=assign typename1=typename1 ...]}

file要载入的模板. assign将载入模板的输出保存到对应的变量中. typename1用户传递给模板的自定义参数. insert和include类似.唯一的区别就是insert载入的模板不会被缓存.


*.if elseif else 不解释.

Smarty变量修饰.

Mar 5th, 2011

今天去了山大.借了3本PHP的书.现在复习下smarty的变量修饰. smarty变量修饰的基本格式是:

{var|modifier_name:parameter1:...}

Smarty常用的变量修饰有: .capitalize 首字母大写. .count_characters:true/false 变量中字符的个数.true计算空格. .cat:”string” 连接字符串. .date_format:”%Y” 按格式输出日期.ex:{$smarty.now|date_format:”%Y:%d:%m”} .default:”string” 变量为空时.输出默认值. .escape:”type” 字符串转码.type可以是HTML,htmlall,URL,quotes,hex,javascript.默认为HTML .lower 转化为小写 .nl2br 和PHP中的nl2br一样. .regex_replace:”reg”:”str” 正则替换 .replace:”str1”:”str2” 替换. .string_format 类似printf .strip 用一个特定字符替换所有重复空白字符 .strip_tags 去掉所有HTML标签 .upper 转化为大写

HTML_QuickForm使用

Mar 4th, 2011

暂且先搜索下HTML_QuickForm类库的资料.这里先记录下吧. HTML_QuickForm是Pear的一个类库.灰常好用. 特点: .可以创建20种常见的表单. .生成兼容性好的XHTML代码. .提供强大和可扩展的表单验证功能. .自动化服务器验证代码(Javascript). .支持文件上传功能. .支持多种模板引擎.

上一篇文章已经有一个简单的例子了.现在来个带有数据验证的例子.

addElement('header', 'header', '请登录');
$form->addElement('text', 'w_Username', '用户名');
$form->addElement('password', 'w_password', '密码');
$form->addElement('submit', 'submit', '提交');
$form->addElement('reset', 'reset', '重置');

$form->addRule('w_Username', '用户名不能为空', 'required');
$form->addRule('w_Username', '用户名必须为3位以上.', 'minlength', 3);
$form->addRule('w_password', '密码不能为空', 'required');

if($form->validate()){
    $form->process('say_hello');
}else{
    $form->display();
}

function say_hello($data){
    print 'Hello, ' . $data['w_Username'];
    print '
';
    print 'Your password is ' . $data['w_password'];
}
?>

这样采用的是服务端验证.你也可以采用客户端验证.只需要在addRule的最后一个参数设置为client就可以了.

Pear的安装及其使用.

Mar 4th, 2011

原来自己编译安装Apache的时候搞过Pear.不过一直都不懂它是干嘛的.只大概知道这是个很大的标准库.从来没想到.它竟然是类似Perl的CPAN. Pear提供好多标准的可供工业生产应用的类库.你可以下载然后直接使用. Pera库基本涵盖了PHP可以操作的任何方面: 1.HTML.表单.分页.模板.等等. 2.GTK.GTK2.这样PHP也可以开发应用程序. 3.IMAGES. 4.DB.MDB.MDB2. 5.XML. 6.MAIL. 7.HTTP.AUTH. 8.MATH. 等等等等.Pear绝对是一个不可缺少的工具. 安装Pear也很简单.windows下运行一个bat.而Linux下是一个php脚本.

wget http://pear.php.net/go-pear.phar
php go-pear.phar

这么简单就安装好了.之后你便可以直接pear install packagename了.可以通过pear list查看当前以安装的库.pear remote-list查看可取得的pear库列表.pear upgrade-all可以升级.pear uninstall packagename可以卸载.

PHPLib_3月4日的不三不四的笔记

Mar 4th, 2011

之前学过了Smarty.不过Smarty更适合网站的开发.而PHPlib也能运用到很多地方.不论是与Smarty的配合还是独立的开发类库什么的.所以这个.也必须得学.还得学好. 首先是手册的问题.教材不多.我下了个中文的简明教程.然后又下了manual.这就是我现在的基本学习方式.中文简明教程+官方manual. manual的话.官网不提供下载.不过没关系.我们用wget下载下来就可以了.

wget -r -nd -np http://www.sanisoft.com/phplib/manual/html/

之后.便可以开始学习了. 使用PHPLIB可以写很多核心函数库和类.用很短的时间开发出很多高效而强大的东西. PHPLIB讲数据库驱动和数据库操作完全分离.这样的好处是你需要的知识仅仅是SQL语言而不用去估计各个不同的数据库间的函数不同.更换数据库时.仅仅需要修改数据库驱动文件.而不许要修改程序中的任何一行. PHPLIB支持session.默认情况下采用了base64加密.为了增强安全性.你也可以自己更改编码方式.同时PHPLIB还将每一个会话都保存到数据库中.因此可以通过查询数据库知道有多少用户在线上. PHPLIB也有很好的权限分级.同时PHPLIB的模板功能也相当不错. PHPLIB安装有两种方式: 1.让PHP解析器加载到内存中.这样安装之后.使用的时候不许要做任何设置.但是当用户无权限操作主机时.比较困难. 2.包含的方式.这个是最简单的.但是相对麻烦.
