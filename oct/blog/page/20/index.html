Linux Kill命令.

Mar 9th, 2011

今天仔细研究了下kill命令. 首先.kill可以杀死jobs进程.也可以杀死jobs外的进程.

kill %1 #杀死第一个job
#此时发送Term信号.
#成功应该看到:
#[1]+ Terminated  
#失败的话:
kill -QUIT %1
#发送QUIT信号.
#成功应该看到:
#[1]+ Exited
#失败的话:
kill -KILL %1
#一个进程不可能躲过KILL信号的.
#进程接受到KILL信号.系统会无条件关闭的.

kill pid可以杀指定pid的进程. 信号与数字之间有关系.

kill -KILL pid
kill -9 pid

killall可以杀死指定进程组的所有进程.

pkill -uroot

杀死指定用户的所有进程.

Bash学习(三).变量.

Mar 9th, 2011

bash默认是文本变量.就是字符串变量.幸好可以通过declare来规定变量类型.-打开选项.+关闭选项.

-a    #数组.
-f    #函数.
-F    #未定义的函数
-i    #整数
-r    #变量只读属性.
-x   #变量导出至环境中.相当于export

当在函数内部使用declare时.相当于local.声明为局部变量. $((statement))可进行数字运算并将结果返回.statement里面的变量加不加$没有关系.里面可以加语句.可以使用关系运算符.关系运算符为普通的><==等.当在条件语句中进行test时.可以使用$((statement))来充当条件.

if [ $a -eq 1 ]; then
     # Code Here
fi
# 等效于
if [ $(($a==1)) ]; then
    # Code Here
fi

算数变量赋值.

let a=3+2   #a=5 但是a不是整数变量
declare -i b=3+2  #b=5且b为整数变量

bash定义数组变量.

name=(apple orange)
name=([0]=apple [1]=asd)

循环读取数组.

for i in "${name[@]}"; do
      echo $i
done

可以通过${#name[@]}来获得数组元素个数.

unset ${name[100]} #销毁下标为100的元素.
unset ${name[@]}    #销毁整个数组.

Bash学习(二).参数捕获

Mar 9th, 2011

命令行参数的捕获方法有好多思路.不过因为不同的用户有不同的参数习惯.所以最好还是通过系统内置的getopts来获取吧. 下面是我写的.通过循环和shift来实现.不过无法解析-ab这种参数.

#!/bin/bash
declare -i argvCount=$#
echo $argvCount
until [ $argvCount -eq 0 ]; do
    case $1 in
        'apple' ) echo 'apple' ;;
        'orange' ) echo 'orange' ;;
    esac
    shift
    argvCount=$argvCount-1
done

接下来是由getopts来实现的版本.

#!/bin/bash
while getopts "ao" opt; do
    case $opt in
        a) echo 'apple'  ;;
        o) echo 'orange' ;;
    esac
done
