明天考试.` 今天把这点内容看完了. 就回寝室. 先接触的Perl.所以后来对php(没接触PCRE之前),python,ruby的正则都有点厌恶.不过接触之后发现,他们的正则也没有想象中的那么差. 首先说下常用的模式语法.

(?:....)                                #匹配.但是不捕获
(?P...)                     #匹配.并存到指定分组中.
(?P=name)                          #匹配一个早期指定的分组所匹配的文本
(?#....)                               #注释.
(?!.....)                               #括号不匹配,开始按括号前内容正则匹配.
                                         #Hello(?!hello)
                                         #hello不匹配,才匹配Hello

额.模式语法有很多.但是一个案例能写出不通思路的正则来,所以说模式语法不一定是必须要掌握的.能解决问题则OK.效率什么的,注意贪婪非贪婪就好. 下面是正则替换字符.

\s                                      #任意空白符
\S                                      #任意非空白字符
\w                                     #任意字母数字.
\d                                      #任意数字.
\D                                     #任意非数字.
\b                                      #单词定界

然后一般情况下为了运行速度,我们不直接采用re.func类用法.而是先把re compile下,然后再正则匹配.

import re

pat = re.compile(str[, flags])

Project2.文档编码转化

May 24th, 2011

这个说不上是project.就是今晚的一个小需求.某个readme无法正常打开.gb2312,gbk,utf8都不成.那我就用chardet检测吧.不废话了.直接代码.

#!/usr/bin/python -tt
# -*- coding: utf-8 -*-

"""\
Usage: python tran.py srcFile [dstFile]
"""
import sys
import chardet

length = len(sys.argv)
if length == 1 or length >3:
    print __doc__
    sys.exit(1)

srcFile = sys.argv[1]
if length == 3:
    dstFile = sys.argv[2]
else:
    dstFile = srcFile+".solved"

with open(srcFile, "r") as f:
    line = f.readline()
    charset = chardet.detect(line)["encoding"]
    f.seek(0,0)

    with open(dstFile, "w") as d:
        while True:
             line = f.readline()
             if not line:
                 break
             line = line.decode(charset).encode("utf-8")
             d.write(line)

urlpatterns = patterns('myFirst.polls.views',

第一个Django程序.

May 17th, 2011

今天开始学习django.时间有限.能学多少是多少.不准备学习Perl的catalyst了.虽然我很喜欢他的LOGO.准备在近期内把django和twisted学了就可以了…开始就多往应用方面发展…OK. 不废话了. 首先是安装django.大多时候我们从源里安装就可以了.其他情况先不考虑了.我们可以通过

import django

来检查django是否安装成功.如果不报错.OK.继续下面的内容. 安装完成后.我们就可以通过django-admin来创建django项目了.

django-admin startproject myFirst

然后django-admin会为我们生成myFirst目录.目录下面有4个文件.

__init__.py
manage.py *
settings.py
urls.py

然后我们可以通过./manage.py runserver来启动django服务器.之后我们便可以在浏览器中localhost:8000来查看我们第一个django程序的效果了. 设置数据库 接下来.我们编辑settings.py.现在我们来修改关于数据库连接方面的配置.以适应你的应用的要求.

DATABASE_ENGINE #我们可以选择mysql或者sqlite3.或者其他的.
DATABASE_NAME    #数据库名称.如果是sqlite3的话.需要是数据库文件的绝对路径.
DATABASE_USER    #sqlite3可以不修改.
DATABASE_PASSWORD #sqlite3可以不修改
DATABASE_HOST    #默认是default.sqlite3可以不用修改.

然后我们可以通过

./manage.py syncdb

来同步数据库.生成数据表. 这里我遇到个问题.默认是没有安装python-mysqldb的.所以安装下.问题不大. 到此.数据库配置完毕..然后我们开始配置模型. 创建模型 这里我们创建一个投票应用.首先创建一个polls模型.

./manage.py startapp polls

然后我们会得到一个polls目录.里面有

__init__.py
models.py
views.py
test.py

我们通过修改models.py来创建两个模型.一个Poll.一个Choice. Poll包含问题以及问题的发布时间. Choice包含选择文本以及投票数. 编辑models.py

from django.db import models

# Create your models here.
class Poll(models.Model):
    question = models.CharField(max_length=200)
    pub_date = models.DateTimeField('date published')

class Choice(models.Model):
    poll = models.ForeignKey(Poll)
    choice = models.CharField(max_length=200)
    votes = models.IntegerField()

如图则创建了两个模型. 需要注意的是,xxxField对应的是数据库中的数据类型.CharField需要指定长度. 然后就是刚才的startapp其实就是创建一个app.app对于django来说是可插拔的.就是说我们可以写好app供其他人使用.所以我们需要把我们写好的app在我们的应用中声明. 返回到上层目录.继续编辑settings.py

#找到Installed App
INSTALLED_APPS = (
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.sites',
    'django.contrib.messages',
    'myFirst.polls'
    # Uncomment the next line to enable the admin:
    # 'django.contrib.admin',
    # Uncomment the next line to enable admin documentation:
    # 'django.contrib.admindocs',
)

然后我们便可以通过

./manage.py sql polls

生成SQL语句.然后继续通过

./manage.py syncdb

同步到应用的数据库中.
