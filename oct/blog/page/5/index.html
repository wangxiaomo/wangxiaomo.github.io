Ubuntu关于默认启动字符端

Oct 23rd, 2011

网上的资料都是骗人的啊…被骗的好苦好苦啊…现在总结下.. First of All.ubuntu没有inittab.ubuntu字符端启动不能通过rcconf.sysv-rc-conf.chkconfig之类的东西改变.所以.这个文章适用于ubuntu… 1.解决方案一.

#查看启动级别.
runlevel
#ub默认返回2
cd /etc/rc2.d
ls | grep gdm
#gdm默认是启动的.启动脚本以K开头.关闭以S开头
mv K20gdm S20gdm

这个方法对于普通服务适用.注意.只是普通服务.gdm不行. 2.解决方案二.

apt-get install sysv-rc-conf
#去掉gdm启动.

原理上和解决方法一相同.但是不适用. 3.解决方案三.

cd /etc/default
vi grub
#找到GRUB_CMDLINE_LINUE
#修改为text
update-grub

这个其实是修改grub以text启动的.. 4.解决方案四.

cd /etc/X11
vi default-display-manager
#修改为false

这个可以…

Emacs_lisp调试.

Oct 20th, 2011

本来决定把lisp放放.语言就是用来用的.但是它对我现阶段的学习没什么太明显用处.就在我准备放弃的时候随手翻了下emacs lisp这本书.然后便改变了想法.决定把这本书快速浏览完再做决定. 都说lisp不需要调试.写完就是写完了.调试的过程发生在书写的过程中,因为不用回过头来debug.所以效率相当高.之前理解的lisp调试就是在一个lisp文件中书写代码,然后在interaction中load进来.写一个函数测试一个.后来发现这个做法虽然可行.但是不符合lisp的风格. 在emacs中,默认有三种模式.emacs-lisp-mode 用于Emacs Lisp代码的编辑（文件名.emacs或后缀.el） lisp-mode 用来编辑另一个Lisp系统的Lisp代码（后缀.l或.lisp） lisp-interaction-mode 用来编辑和运行Emacs Lisp代码（交互模式）.默认进来的时候缓冲区是scratch.这个就是lisp-interaction-mode.我们只讲这个. 在这里书写lisp代码.在结束的时候c-x x-e便可以进行调试观察.如果没有发生错误则继续编写.一边编辑一边调试.这样的感觉确实很爽.(ps.c-u c-x c-e可以把结果输出到缓冲区)

Lisp小窥

Oct 19th, 2011

最近是实验+考试周。没有大块的时间用来看算法方面的知识了。所以就找点时间看看其他的。在豆瓣上看到了http://www.douban.com/note/178148141/感觉lisp写起来还是挺带感的。而且函数式编程据说对程序员的自身发展挺重要的。那我就看看吧。 学了一天之后才决定写文章的。所以之前看的那些东西就不在赘述了。现在开始读书笔记。 1.function return values

(defun foo(n)
  (return-from foo n))

2.循环.

(defun foo(i j step)
  (loop for x from i to j by step do
    (format t "Hello~%")))

3.let用法.

(defun foo(x)
  (format t "Parameter:~a~%" x)
  (let ((x 2))
    (format t "Outer LET:~a~%" x)
    (let ((x 3))
       (format t "Inner LET:~a~%" x))
    (format t "Outer LET:~a~%" x))
  (format t "Parameter:~a~%" x))

4.defvar和defparameter.

;defparameter 不管var是否存在.强制初始化.
;defvar 如果var存在.则不做任何改变.
(defparameter *count* 30)
(format t "~a%" *count*)
(defvar *count* 10)
(format t "~a%" *count*)

5.let优先于局部.defvar,defparameter定义普通变量.setq,setf直接修改内存. ..参考http://wiki.ubuntu.org.cn/Common_LISP_Hints 其他的暂时不写了.学习这个感觉靠的还是感觉…

系统

uname -a # 查看内核/操作系统/CPU信息

env # 查看环境变量

资源

free -m # 查看内存使用量和交换区使用量

grep MemTotal /proc/meminfo # 查看内存总量

grep MemFree /proc/meminfo # 查看空闲内存量

uptime # 查看系统运行时间、用户数、负载

cat /proc/loadavg # 查看系统负载

磁盘和分区

mount | column -t # 查看挂接的分区状态

fdisk -l # 查看所有分区

swapon -s # 查看所有交换分区

hdparm -i /dev/hda # 查看磁盘参数(仅适用于IDE设备)

dmesg | grep IDE # 查看启动时IDE设备检测状况

网络

ifconfig # 查看所有网络接口的属性

iptables -L # 查看防火墙设置

route -n # 查看路由表

netstat -lntp # 查看所有监听端口

netstat -antp # 查看所有已经建立的连接

netstat -s # 查看网络统计信息

进程

ps -ef # 查看所有进程

top # 实时显示进程状态

用户

w # 查看活动用户

id <用户名> # 查看指定用户信息

last # 查看用户登录日志

crontab -l # 查看当前用户的计划任务

服务

chkconfig –list # 列出所有系统服务

重写数据库连接库.

Oct 14th, 2011

之前一直用着郑叶平学长封装好的库与数据库进行交互.后来用着发现许多基本的需求都没有很好的解决.晚上的时候对sae的sql文档进行了仔细阅读.发现里面一些功能我需求也不是很大.仔细思考过去曾经的应用.重新封装了下. 基本的需求: 1.增. 返回id 2.删改. 返回受影响的行数. 3.查. 返回结果. 4.后续需求再加. 目录结构如下： —–+SQLParser ——– init.py ——– var.py ——– parser.py ——– mysql_parser.py ——– sqlite_parser.py

基本代码如下： 1.var.py 一些基本的配置信息.

#-*- coding: utf-8 -*-

EXCEPTION = '========> Exception Caught Here <========='

DEFAULT_SQLITE_DB = 'default.db'

DEFAULT_MYSQL_HOST = 'localhost'
DEFAULT_MYSQL_USER = 'root'
DEFAULT_MYSQL_PASS = '123'
DEFAULT_MYSQL_DB = 'default'

2.parser.py 虚拟类.

#-*- coding: utf-8 -*-
import traceback
import var

class Parser(object):
    """
    abstract class
    """
    def __init__(self):
        self.conn = None
        self.cursor = None

    def __del__(self):
        try:
            self.cursor.close()
            self.conn.close()
        except Exception as e:
            print var.EXCEPTION
            traceback.print_exc()

    def commit(self):
        self.conn.commit()

    def insert(self, sql):
        try:
            self.cursor.execute(sql)
        except Exception as e:
            print var.EXCEPTION
            traceback.print_exc()
        return self.cursor.lastrowid

    def delete(self, sql):
        try:
            self.conn.execute(sql)
            self.commit()
        except Exception as e:
            print var.EXCEPTION
            traceback.print_exc()
        return self.conn.total_changes

    def update(self, sql):
        return self.delete(sql)

    def select(self, sql):
        try:
            self.cursor.execute(sql)
        except Exception as e:
            print var.EXCEPTION
            traceback.print_exc()
        return self.cursor.fetchall()

if __name__ == '__main__':
    import sys
    sys.exit(0)

3.mysql_parser.py

#-*- coding: utf-8 -*-

import MySQLdb

from var import DEFAULT_MYSQL_HOST, \
    DEFAULT_MYSQL_USER, \
    DEFAULT_MYSQL_PASS, \
    DEFAULT_MYSQL_DB
from parser import Parser

class MySQL(Parser):
    def __init__(self,
            host=DEFAULT_MYSQL_HOST,
            user=DEFAULT_MYSQL_USER,
            passwd=DEFAULT_MYSQL_PASS,
            db=DEFAULT_MYSQL_DB):

        Parser.__init__(self)
        self.conn = MySQLdb.connect(host=host,
            user=user,
            passwd=passwd,
            db=db);
        self.cursor = self.conn.cursor()

if __name__ == '__main__':
    import sys
    sys.exit(0)

4.sqlite_parser.py

#-*- coding: utf-8 -*-

import sqlite3

import var
from parser import Parser

class SQLite(Parser):
    def __init__(self, db_path=None):
        Parser.__init__(self)
        if not db_path:
            print "Database Not Specified, Use Default"
            print "Default DB: ", var.DEFAULT_SQLITE_DB
            self.conn = sqlite3.connect(var.DEFAULT_SQLITE_DB)
        else:
            self.conn = sqlite3.connect(db_path)
        self.cursor = self.conn.cursor()

if __name__ == '__main__':
    import sys
    sys.exit(0)
