GeoFix.火星坐标修改 。 修正。

Dec 3rd, 2011

前几天写得那个geofix最终没有派上用处。因为在google earth中不经过修正的坐标最终正确的显示了。这是一个可喜可贺的消息啊。因为geofix之后的坐标也很诡异。如果任何途径都没法解决的话，最终我们就只能通过对范围区域内手动进行修正了。 今天老周让做gmaps的版本，恩。google maps v3版本的api很强力，使用起来很简单，载入速度的话，也很快。毕竟v3官方就说明了推荐移动终端使用，对整个处理引擎做出了优化。ps.这里对载入速度快做下解释。官方的api是maps.google 这个载入速度受GFW的影响。所以我的好基友就随便改成了ditu.google.com。恩。中国用户你懂得。可是速度还是比较慢。后来上ditu.google看了下，发现域名是cn.所以改成ditu.google.cn。速度蹭蹭的。。。 之后。在c#中生成json path.然后使用web browser打开。可是。可是。可是。路径偏移了。所以就又得校正。果断使用自己写的geofix.谁知速度那是相当的慢。仔细思考了下，发现为了使自己的类库可是直接静态引用，所以在每次转换的时候都重新读取xml.所以大量的时间消耗在io上。所以果断重写。

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Xml;
 
namespace GeoFix{
    class GeoFix{
        public static string sxml = "sxml.xml";
        public static string dxml = "dxml.xml";
        public XmlDocument xmlDoc;
        public XmlNodeList snodes;
        public XmlNodeList dnodes;
 
        // Geo Fix. 
        // 修正火星坐标.
 
        public GeoFix(){
            xmlDoc = new XmlDocument();
            xmlDoc.Load(sxml);
            snodes = xmlDoc.GetElementsByTagName("data");
            xmlDoc.Load(dxml);
            dnodes = xmlDoc.GetElementsByTagName("data");
        }

        public  double[] fix(double lat, double lng){
            Double[] ret = refix(lat, lng, 0);
            if(ret[0] == 0.0 && ret[1] == 0.0){
                ret = refix(lat, lng, 1);
            }
            return ret;
        }
 
        public  double[] refix(double lat, double lng, int type){
            Double[] ret = new Double[2]{0.0, 0.0};
            XmlNodeList nodes;
            if (type == 0){
                nodes = snodes;
            }else{
                nodes = dnodes;
            }
            foreach(XmlNode node in nodes){
                XmlNodeList childNodes = node.ChildNodes;
                double xla = double.Parse(childNodes[0].InnerText);
                double xlo = double.Parse(childNodes[1].InnerText);
                double dla = double.Parse(childNodes[2].InnerText);
                double dlo = double.Parse(childNodes[3].InnerText);
                double la = double.Parse(childNodes[4].InnerText);
                double lo = double.Parse(childNodes[5].InnerText);
                if(xlalat && xlolng){
                    ret[0] = lat-la;
                    ret[1] = lng-lo;
                    break;
                }
            }
            return ret;
        }
    }
}

GeoFix. 火星坐标修改.

Nov 30th, 2011

今天gps传回的数据在google earth中有很大的误差.老周说是经纬度要校验后再显示,突然就想起了前阵子做looking4u时候看到的火星坐标的问题.饿.出了这个问题没有第一时间想起来这个问题,my fault. 火星坐标是为了保护国家安全信息的…火星坐标与标准地理坐标的转化也比较麻烦. 据CSDN说,转化过程是这个样子: 火星坐标除以∏，再开8次方，再以e为底求一次对数，再展开为傅立叶级数，取其中奇数项常量做卷积，可得相应地球坐标。 不过我觉得这个比较不靠谱. 火星坐标的转化公式也是每过一段时间就变化的.所以需要找一个简单而比较犀利的方法. 方法一.http://www.anttna.com/goffset/goffset1.php?lat=纠偏前的纬度值&lon=纠偏前的经度值. 恩.这个方法比较简单. 方法二.先下载两个xml文件.SXML.xml和DXML.xml文件.然后运行如下代码:

using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Xml;

namespace GeoFix{
    class GeoFix{
        public static string sxml = "sxml.xml";
        public static string dxml = "dxml.xml";


        // Geo Fix. 
        // 修正火星坐标.

        public static double[] fix(double lat, double lng){
            Double[] ret = refix(lat, lng, 0);
            if(ret[0] == 0.0 && ret[1] == 0.0){
                ret = refix(lat, lng, 1);
            }
            return ret;
        }

        public static double[] refix(double lat, double lng, int type){
            Double[] ret = new Double[2]{0.0, 0.0};
            XmlDocument xmlDoc = new XmlDocument();
            if(type == 0){
                xmlDoc.Load(sxml);
            }else{
                xmlDoc.Load(dxml);
            }
            XmlNodeList nodes = xmlDoc.GetElementsByTagName("data");
            foreach(XmlNode node in nodes){
                XmlNodeList childNodes = node.ChildNodes;
                double xla = double.Parse(childNodes[0].InnerText);
                double xlo = double.Parse(childNodes[1].InnerText);
                double dla = double.Parse(childNodes[2].InnerText);
                double dlo = double.Parse(childNodes[3].InnerText);
                double la = double.Parse(childNodes[4].InnerText);
                double lo = double.Parse(childNodes[5].InnerText);
                if(xlalat && xlolng){
                    ret[0] = lat-la;
                    ret[1] = lng-lo;
                    break;
                }
            }
            return ret;
        }
    }
}

两个方法都可以进行校验,不过效果.恩..我觉得一般…有待后续改进..

Git使用`

Nov 25th, 2011

首先想要说，svn用户转git内心有种排斥的心里啊… 不废话了。 说配置。

配置git. git的配置.git的配置分为三种: 1.系统性配置 git config –system. 配置文件在/etc/gitconfig 2.用户性配置 git config –global.配置文件在~/.gitconfig 3.工程性配置 git config 配置文件在工程目录.git/config. 配置用户信息。每次push的时候会把用户信息push上去。所以必须配置

git config user.name "codevvip"
git config user.email "wxm4ever@gmail.com"

配置默认的文本编辑器。非必须。

git config core.editor vim

配置差异分析工具。非必须

git config merge.tool vimdiff

检查当前配置。

git config --list

配置项目。 初始化git.初始化之后当前目录会出现.git目录。

git init

从现有仓库克隆。git clone. 注意这里是clone而不是svn里的checkout.clone会把所有版本返回。而checkout只返回一个特定的版本。

git clone git://github.com/wxmvvip/code.git

clone下来为所有的版本库。以后的git add rm commit等操作都是基于本地的。如果想要共享的话，需要push一下`

Svn 基本命令.[粘贴]

Nov 14th, 2011

svn log svn://127.0.0.1/zxProject/ –username xxxxxx –password xxxxxx |grep “2009-05-29 16:38*” [xx@wap01 xx]$ svn log -v -r134 svn://127.0.0.1/zxProject/ –username xxxxxx –password xxxxxx [xx@wap01 xx]$ svn log -v -r134:135 svn://127.0.0.1/zxProject/ –username xxxxxx –password xxxxxx

svn 命令

1、将文件checkout到本地目录 svn checkout path（path是服务器上的目录） 例如：svn checkout svn://192.168.1.1/pro/domain 简写：svn co

2、往版本库中添加新的文件 svn add file 例如：svn add test.php(添加test.php) svn add *.php(添加当前目录下所有的php文件)

3、将改动的文件提交到版本库 svn commit -m “LogMessage“ [-N] [–no-unlock] PATH(如果选择了保持锁，就使用–no-unlock开关) 例如：svn commit -m “add test file for my test“ test.php 简写：svn ci

4、加锁/解锁 svn lock -m “LockMessage“ [–force] PATH 例如：svn lock -m “lock test file“ test.php svn unlock PATH

5、更新到某个版本 svn update -r m path 例如： svn update如果后面没有目录，默认将当前目录以及子目录下的所有文件都更新到最新版本。 svn update -r 200 test.php(将版本库中的文件test.php还原到版本200) svn update test.php(更新，于版本库同步。如果在提交的时候提示过期的话，是因为冲突，需要先update，修改文件，然后清除svn resolved，最后再提交commit) 简写：svn up

6、查看文件或者目录状态 1）svn status path（目录下的文件和子目录的状态，正常状态不显示） 【?：不在svn的控制中；M：内容被修改；C：发生冲突；A：预定加入到版本库；K：被锁定】 2）svn status -v path(显示文件和子目录状态) 第一列保持相同，第二列显示工作版本号，第三和第四列显示最后一次修改的版本号和修改人。 注：svn status、svn diff和 svn revert这三条命令在没有网络的情况下也可以执行的，原因是svn在本地的.svn中保留了本地版本的原始拷贝。 简写：svn st

7、删除文件 svn delete path -m “delete test fle“ 例如：svn delete svn://192.168.1.1/pro/domain/test.php -m “delete test file” 或者直接svn delete test.php 然后再svn ci -m ‘delete test file‘，推荐使用这种 简写：svn (del, remove, rm)

8、查看日志 svn log path 例如：svn log test.php 显示这个文件的所有修改记录，及其版本号的变化

9、查看文件详细信息 svn info path 例如：svn info test.php

10、比较差异 svn diff path(将修改的文件与基础版本比较) 例如：svn diff test.php svn diff -r m:n path(对版本m和版本n比较差异) 例如：svn diff -r 200:201 test.php 简写：svn di

11、将两个版本之间的差异合并到当前文件 svn merge -r m:n path 例如：svn merge -r 200:205 test.php（将版本200与205之间的差异合并到当前文件，但是一般都会产生冲突，需要处理一下）

12、SVN 帮助 svn help svn help ci

13、版本库下的文件和目录列表 svn list path 显示path目录下的所有属于版本库的文件和目录 简写：svn ls

14、创建纳入版本控制下的新目录 svn mkdir: 创建纳入版本控制下的新目录。 用法: 1、mkdir PATH… 2、mkdir URL… 创建版本控制的目录。 1、每一个以工作副本 PATH 指定的目录，都会创建在本地端，并且加入新增 调度，以待下一次的提交。 2、每个以URL指定的目录，都会透过立即提交于仓库中创建。 在这两个情况下，所有的中间目录都必须事先存在。

15、恢复本地修改 svn revert: 恢复原始未改变的工作副本文件 (恢复大部份的本地修改)。revert: 用法: revert PATH… 注意: 本子命令不会存取网络，并且会解除冲突的状况。但是它不会恢复 被删除的目录

16、代码库URL变更 svn switch (sw): 更新工作副本至不同的URL。 用法: 1、switch URL [PATH] 2、switch –relocate FROM TO [PATH…] 1、更新你的工作副本，映射到一个新的URL，其行为跟“svn update”很像，也会将 服务器上文件与本地文件合并。这是将工作副本对应到同一仓库中某个分支或者标记的 方法。 2、改写工作副本的URL元数据，以反映单纯的URL上的改变。当仓库的根URL变动 (比如方案名或是主机名称变动)，但是工作副本仍旧对映到同一仓库的同一目录时使用 这个命令更新工作副本与仓库的对应关系。

17、解决冲突 svn resolved: 移除工作副本的目录或文件的“冲突”状态。 用法: resolved PATH… 注意: 本子命令不会依语法来解决冲突或是移除冲突标记；它只是移除冲突的 相关文件，然后让 PATH 可以再次提交。

18、输出指定文件或URL的内容。 svn cat 目标[@版本]…如果指定了版本，将从指定的版本开始查找。 svn cat -r PREV filename > filename (PREV 是上一版本,也可以写具体版本号,这样输出结果是可以提交的)

多文件工程. test.cpp [main所在] func.h func.cpp

target = main
objects = test.o func.o
sources = test.cpp func.cpp

$(target):$(objects)
    g++ -o $(target) $(objects)

$(objects):$(sources)
    g++ -c $(sources)

.PHONY:clean
clean:
    rm -rf $(objects) $(target)

然后在终端下

make
#清理
#make clean

就可以了
