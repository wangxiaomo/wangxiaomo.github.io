编码终极解决方案

May 29th, 2011

一直纠结于编码, 今天终于找到了点门路… 字符串在Perl或者Python内部都是unicode的形式,因此,在做编码转化的时候通常以unicode为中间编码,然后通过encode来转化为特定的编码.即先探测字符串编码(chardet,Encode::Guess),然后decode掉,成为unicode编码,然后再通过encode来转化编码…


然后,要注意的是,代码中的字符串与文件的编码一致.所以说Python中要想使字符串为unicode而不受文件编码的影响,需要用u”来定义. 如果对unicode进行解码则会发生错误,所以我们需要先判断.以Python举例/

import chardet
s = chardet.detect(str)
if s.encoding == "utf-8":
    pass

isinstance(s, unicode)

我们可以通过sys模块来获得系统默认编码.

import sys
print sys.getdefaultencoding()

多进程和多线程的本质区别.[py]

May 25th, 2011

之前一直不是很明白这两者的具体区别,感觉用哪个无所谓,看自己的心情,windows下的话thread相对简单,linux下清一色的fork.其实这样的做法是错误的… 线程运行在创建它的进程内部,他们共享所有的数据和系统资源. 下面看测试代码..

import os,threading
import time,sys

data = [1,2,3]
mutex = threading.Lock()

pid = os.fork()
if pid == 0:
    with mutex:
        print "Sub:", data[1]
        data[1] = 3
        print "Sub:", data[1]
        sys.exit(0)

time.sleep(3)
print "Main:", data[1]

根据上面这个代码,我们可以知道在fork,也就是多进程编程中,会把所有资源复制一遍,相互不影响. 接下来看多线程的. 为了方便,我们混用thread和threading模块.

import thread,threading
data = [1,2,3]
mutex = threading.Lock()

def change():
    global data, mutex
    with mutex:
         data.append(4)

thread.start_new_thread(change, ())
thread.start_new_thread(change, ())

for i in data:
    print i

OK.

Python_optparse模块

May 25th, 2011

python处理sys.argv的方式有很多,最常见的就是getopt和optparse. 文档上推荐optparse,因为它更加清晰,方便.. 下面直接上代码.

import optparse
p = optparse.OptionParser()

#简单的选项,不带参数,为bool类型
p.add_option("-t", action="store_true", dest="tracing")

#接受字符串参数
p.add_option("-o", "--outfile", action="store", type="string", dest="outfile")

#接受整数参数
p.add_option("-d", "--debuglevel", action="store", type="int", dest="debug")

#带有一些选择的选项
p.add_option("--speed", action="store", type="choice", dest="speed",
                  choices=["slow", "fast", "ludicrous"])

p.set_defaults(tracing=False,
                    debug=0,
                    speed="fast"
)

opt,args = p.parse_args()

注意,action里面需要指定storexxx的值,store用来表明需要被存储,store_true表明出现则为true.store_false表明出现则为false
