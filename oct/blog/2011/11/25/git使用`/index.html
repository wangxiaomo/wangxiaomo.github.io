Git使用`

Nov 25th, 2011

首先想要说，svn用户转git内心有种排斥的心里啊… 不废话了。 说配置。

配置git. git的配置.git的配置分为三种: 1.系统性配置 git config –system. 配置文件在/etc/gitconfig 2.用户性配置 git config –global.配置文件在~/.gitconfig 3.工程性配置 git config 配置文件在工程目录.git/config. 配置用户信息。每次push的时候会把用户信息push上去。所以必须配置

git config user.name "codevvip"
git config user.email "wxm4ever@gmail.com"

配置默认的文本编辑器。非必须。

git config core.editor vim

配置差异分析工具。非必须

git config merge.tool vimdiff

检查当前配置。

git config --list

配置项目。 初始化git.初始化之后当前目录会出现.git目录。

git init

从现有仓库克隆。git clone. 注意这里是clone而不是svn里的checkout.clone会把所有版本返回。而checkout只返回一个特定的版本。

git clone git://github.com/wxmvvip/code.git

clone下来为所有的版本库。以后的git add rm commit等操作都是基于本地的。如果想要共享的话，需要push一下`
