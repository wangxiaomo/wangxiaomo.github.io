Android 数据存储

Jun 25th, 2011

Android提供了如下几种存储数据的方式:Preference(配置),File(文件),SQLite和网路. 1.Preference是一种轻量级的存取数据方式,用到了SharedPreferences.

//读取数据.
import android.content.SharedPreferences;

SharedPreferences pre = getSharedPreferences(TEMP_SMS, MODE_WORLD_READABLE);
String content = pre.getString("sms_content","");
myEditText.setText(content);

//存储数据.
SharedPreferences.Editor editor = getSharedPreferences(TEMP_SMS, MODE_WORLD_WRITEABLE).edit();
editor.putString("sms_content", myEditText.getText().toString());
editor.commit();

2.我们可以将一些数据以文件的形式保存到设备中.

import java.io.FileInputStream;
import java.io.FileOuputStream;

//封装成方法.
private String read(){
    try{
        //实例化文件输入流对象.
        FileInputStream fis = openFileInput(FILE_NAME);
        byte[] buffer = new byte[fis.available()];
        fis.read(buffer);
        return new String(buffer);
    }catch(Exception e){
        e.printStackTrace();
    }
    return null;
}
private void write(String content){
    try{
        FileOutputStream fos = openFileOutput(FILENAME, MODE_APPEND);
        fos.write(content.getBytes());
        fos.close();
    }catch(Exception e){
        e.printStackTrace();
    }
}

3.SQLite数据库.SQLite不像其他数据库,它没有服务器进程,所有的内容包含在同一个单文件中.该文件是跨平台的,可以自由复制.所以SQLite在嵌入式领域得到了广泛的应用. 打开或者创建数据库.

SQLiteDatabase.openOrCreateDatabase("/data/data/document/database/temp.db", null);

创建表.

private void createTable(SQLiteDatabase db){
    String sql = "CREATE TABLE UserTbl(_id integer primary key 
autoincrement, username text, password text)";
    db.execSQL(sql);
}

插入数据.两种方式,一种是调用insert方法,一种是构造SQL语句,然后execSQL.这里说明下insert方法.

private void insert(SQLiteDatabase db){
    ContentValues cv = new ContentValues();
    cv.put("username", "wxm");
    cv.put("password", "123");
    db.insert("UserTab", null, cv);
}
//insert的第二个参数表示空列的默认值.

private void update(SQLiteDatabase db){
    ContentValues values = new ContentValues();
