Smarty基本安装配置.

Feb 9th, 2011

Smarty 使PHP脚本从设计中独立出来. 在应用程序的开发流程中:在提交计划文档之后.界面设计者(美工)制作了网站的外观模型,然后把它交给后台程序员.程序员使用PHP实现商业逻辑.同时使用外观模型做成基本架构.然后工程被返回到html页面设计者继续完善.这样工程可能在后台程序员和页面设计者之间来来回回好几次.因此.一个好的模版支持就显得特别重要. smarty不尝试将逻辑与模版完全分离.而是尽可能的远离.使得内容更容易管理.程序更容易升级 Smarty安装: 1>Smarty3.x PHP5.2+ 2>Smarty2.x PHP4 or 5 Smarty使用一个SMARTY_DIR的php常量作为它的系统库目录.如果没有在php.ini中的include_path里手动添加Smarty.class.php的话.必须手动配置SMARTY_DIR. 配置好之后便可以创建Smarty实例.

ps:我按照文档所说的配置了php.ini但是不成功.所以我直接require的时候指明了路径.也可以按如下方法使用.感觉更加灵活些.

smarty要求4个基本目录.templates.templates_c.configs.cache.可以修改变量来改变目录的名称.在Smarty.class.php里面设置. Smarty的$compile_dir和$cache_dir必须可写.也就是说templates_c和cache目录必须可写.可以chmod 770.让用户和组有写权利.其他用户最好无权限操作. 1.创建index.tpl让smarty载入.

{* Smarty *}
Hello, {$name}!

其中{ }中的东西是模版注释.不是必须的. 2.修该根目录下的index.php

template_dir = './templates/';
$smarty->compile_dir = './templates_c/';
$smarty->config_dir = './configs/';
$smarty->cache_dir = './cache/';

$smarty->assign('name', 'XiaoMo');
$smarty->display('index.tpl');
?>

好的.现在开始.Smarty的基本设置已经完成.

更灵活的配置方法.我们可以在根目录下新建一个setup.php.

Smarty();
        $this->template_dir = './templates/';
        $this->compile_dir = './templates_c/';
        $this->config_dir = './configs/';
        $this->cache_dir = './cache/';
        $this->caching = true;
        $this->assign('app_name', 'Smarty_Test');
    }
}
?>

然后适当修改index.php.

assign('name', 'XiaoMo');
$smarty->display('index.tpl');
?>
