Xdebug折腾日记

Mar 10th, 2011

今天上午要写一个web services的项目.wsdl位于远端服务器上.但是有错误.悲剧.try_catch没有捕获到任何异常.但是页面无任何输出.__getFunctions也没有输出.彻底悲剧. 然后就准备调试调试这个页面. 首先是log.查看了半天也没什么错误.果断放弃.用Netbeans调试的话.说实话.没试过.还得配置一堆服务器信息.我就放弃了. 后来上网查下有没有简单点的东西.FireBug和FirePHP.不过我还是没调试出来.可能是这个调试方法对普通用户来说还是太过于麻烦.毕竟需要包含库文件.然后调用函数才行. 那就用xdebug吧.wamp自带的这个调试起来不错.可是.我在linux下没装. 哎.装 上网下了源码.发现没有configure文件.然后直接make吧.也不行.果断就看README了.

apt-get install php5-dev
phpize
./configure
make

之后变会在module下生成so文件.然后在php.ini中引用就可以了. 可惜.上面是理论的mannual流程.我在configure那里就出错了.google了半天也没查到错误.可能是错误太简单了.语法问题.不过我凭借我那点bash知识看了半天也没看出来哪有问题. 放弃. 换种方式.google到.

apt-get install php5-dev php5-pear
pecl install xdebug

这里不得不说下pecl和pear了.都是php的拓展.有人说pecl是底层实现.pear是上层实现.不清楚. 然后在php.ini中增加拓展. 开始测试.phpinfo中xdebug成功增加.然后继续测试.

查看页面没有错误.哎.有问题啊.. 解决了半天.未果. 于是来到开拓者继续测试.

首先下载了apache2.2.然后安装.结果这个破网吧80端口还被绑定.而且hosts文件也没有localhost到127.0.0.1的映射.哎.没办法.只能http://127.0.0.1:8080测试了. 然后下载了php5.3 vc9 no secury thread installer版本.安装完以后在apache的httpd.conf中增加LoadModule把php5.dll加载进去.结果不成功.记忆中需要下载压缩包版本的.于是重新下载.不行.上网一查.需要php5apache2.dll. 然后又搜索了半天发现需要vc6版本的.然后我又下载了vc6的no secury thread版本.总算有个php5apache.dll了.然后继续Load还不行.上网再查.必须是php5apache2.dll.然后我又查查这个dll哪获得.结果发现需要vc6的secury thread版本.于是继续下载.这次确实有了.加载以后.还是不成功.我又仔细观察了下文件列表.发现还有个php5apache2_2.dll.这个是不是说apache版本的.(自己下载的是2.2的).尝试下竟然成功了. 这里再说下php在apache中以cgi形式运行的方法.修改配置文件如下:

AddType application/x-httpd-php .php
Alias application/x-httpd-php php-cgi.exe

我测试了下.不行.不清楚为啥.完了再找原因吧. 然后就安装xdebug.windows直接下载dll就行. 其他的就不描述了.这里说下重点. 调试.注意多查看phpinfo
