CSS定位需要注意的一些问题.

Mar 21st, 2011

今天早上升国旗…6点半就起床了. 还真是要命. 既然起来了就来实验室吧. 昨晚还没给我的64MB ub server装上fluxbox了. 来了整了一会.发现没什么进展.因为在vbox下ubuntu提示不能build kernel module.以前装arch的时候遇到过.解决方案是:

pacman -S kernel26-headers

所以我果断.

aptitude install kernel26-headers

结果提示没有该包.然后我又aptitude search了半天.没装X.还得more查看.累死. 不过还是没有.就google了.还是没进展. 然后阿娇姐问我我的代码在win下不能运行.我就搞他去了.============>结果是编码的问题.在linux环境下编译的.UTF8编码.放到VC6下就不行了. 然后我就准备改下主页的主题. 首先下了F2.最后害怕opensalon的人说我抄袭.我就换了.找到一个比较给力的主题.于是马上准备动手hack之.结果打开后发现被加密了.算了.人家也不容易.就不要找解密软件hack了.再换一个. 想找个朴素点的.就找了网上一个MiniWp主题.单纯的用CSS写的. 然后还是有些地方不符合我的想法.继续HACK. 他的CSS代码我改了半天也不行.定位老是错误.但是我自己写的测试是没问题的.我就奇怪了.

然后我就rename style.css style.css.bak然后自己重写css layout. 期间发现了几个问题: 1.一般定位float足够了.根本不许要加什么position.除非为了实现广告的效果.用到postion:fixed 2.还有就是两列三列模式什么的.单纯的float:left就可以了.但是我今天发现了footer错位的问题.改了半天还是不合适.结果是因为没有clear掉float. 3.外层DIV的宽度必须大于等于内部多个float的div的宽度和.不然内部DIV会移动到下一层.不是同层效果了.所以要控制好宽度.如果实在有问题的话,那就配合padding和oveflow来实现吧. 大概就是这么多….下午还有毛邓… fighting..!~
