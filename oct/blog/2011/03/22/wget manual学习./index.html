Wget Manual学习.

Mar 22nd, 2011

继续上篇日志.都说了.网上的那群人写的东西不是错的.就是最简单的东西.所以我决定看看wget的manual.来挖掘下社交网络的主角是怎么用–post-data这个参数的. 首先.

man wget
# Wget - The non-interactive network downloader
# 非交互式下载工具.

wget支持许多种协议.HTTP,HTTPS,FTP,以及支持代理的上述协议.这样已经够了.夫付何求?wget是非交互式下载工具.非交互的理解就是可以在后台执行,尽管用户没有log on.&可以使程序后台执行.但是必须保持SHELL开启.也就是说.用户登出后就会停止运行.所以需要配合nohup来使用.

nohup command &

这样就可已保证是non-interactive的进程了. wget可以跟踪HTML,XHTML,CSS中的链接地址.进而来产生本地镜像.wget默认情况下遵守robot.但是可以通过-e来跳过. 语法介绍: wget是通过GNU getopt来获取参数的.所以可以支持长参数也可以支持短参数.

wget -r --tries=10 http://fly.srk.fer.hr/ -o log
# 递归下载网页.每次链接至多重试10次.并把日志输出到log文件中.

-o log也可以写成-olog. wget可以设置不下载的目录.可以写在/etc/wgetrc也可以写在~/.wgetrc中.当然也可以通过参数-X来指定.

wget -X '' -X /~nobody,/~somebody
# -X ''可以指定跳过wgetrc的限制.
# 然后再通过-X来指定.
# 多个目录之间用,分割

基本选项:

-V
--version
# 查看wget的版本信息.
-h
--help
# 查看帮助信息.
-b
--background
# 后台运行.
-e command
--execute command
# 执行配置命令.
-o logfile
--output-file=logfile
# 输出log信息.
-a logfile
--append-file=logfile
# 追加log到logfile中.
-d 
--debug
# 开启调试模式.
-q
--quiet
# 安静模式.没有输出.与-nv --no-verbose同意.
-v
--verbose
# 默认开启.冗杂模式.有输出.
-i file
--input-file=file
# 按照指定的文件来下载内容.
-t number
--tries=number
# 重试次数.
-c
--continue
# 支持断点下载.

以上都是最基本的用法.下面来比较高级的.

--user=user
--password=password
# 可以指定FTP/HTTP访问所需要的认证帐号密码.
# 可以具体到--ftp-user/--http-user
--ask-password
# 那种每次链接都需要密码验证的链接.
# -password不能使用的时候用这个测试下.
--local-encoding=encoding
--remote-encoding=encoding
# 用来指定编码.
--header=heade-line
# 在每次请求时发送头.
--save-headers
# 保存请求头.和相应头
--post-data=string
--post-file=file
# 使用POST方式来发送信息.
--keep-session-cookies
# 保存session和cookies信息.
# 这个需要配合--save-cookies和--load-cookies来使用.
# keep这个只是保证session不断.至于cookie什么的还需要自己操作.
