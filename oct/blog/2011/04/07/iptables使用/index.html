Iptables使用

Apr 7th, 2011

今天在宿舍连接实验室的机子.悲剧了. 连接不上.当时那个尴尬啊.[能够PING通.但是SSH和FTP都不能用].要不是因为下午有课.我就直接飞奔到实验室大干一场了.

service sshd status
service vsftpd status
chkconfig | grep -E 'sshd|vsftpd'

检查一切正常.我就无语了.然后我就在本地进行测试. 终端下ssh和ftp一切正常.然后我到浏览器中访问ftp.发现fedora提示selinux警告信息.当时我就明白了.防火墙没有关…. 之前用的发行版貌似默认都没有开启这些.然后我就仔细排查了下.在X下进行了相关改动.毕竟终端下的配置还不是很了解.这不看不要紧. 一看就发现fedora默认开启selinux和iptables. selinux貌似就是种安全策略吧.所以先不管它.先学习下iptables. iptables是一种在内核层面上的防火墙.可以通过检查数据包包头来进行拦截等.iptables默认有三个表.默认是filter表.我们针对这个表进行操作就可以了. 接下来直接说iptables的语法吧.

iptables [-t table] COMMAND MATCH TARGET

一般情况下我们操作的都是filter表.所以-t选项基本不怎么用了. 然后就是COMMAND部分.一般常用

-L #列出表内filter
-I  #插入到指定位置.
-A #插入Append
-D #删除指定
-F #全部删除

filter表下默认有三条链路.INPUT.OUTPUT.FORWARD.我们需要在COMMAND中指定操作链路.例如:

iptables -I INPUT 1 -s 172.29.143.87 -j REJECT

COMMAND后面就是MATCH条件了.一般常用的就是-s[来源地址]-d[目的地址].–sport[来源端口].–dport[目的端口].-p[协议]. 最后就是TARGET.需要由-j 进行引导.一般常用的有

ACCEPT
DROP
REJECT
RETURN

DROP的话不进行数据包的处理.REJECT返回拒绝信息.所以具体使用取决实际情况.建议DROP`
