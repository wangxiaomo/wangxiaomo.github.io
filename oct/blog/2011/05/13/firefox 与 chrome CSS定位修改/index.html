Firefox 与 Chrome CSS定位修改

May 13th, 2011

本来想搞定wp_nonce_url就继续python了.` 结果中途想起了成昊同学说我的页面在chrome下显示不正常.所以我就下载了chromium-browser测试了下. 嘿. 还真是错的. 我当时就纠结了. 仔细观察了下. 发现是content和sidebar定位错误.由于好长时间没有用chromium了.所以也不记得css测试的插件了. 所以就到ff下查看了下css格局. 由于css不是自己写的.所以理清楚他的结构还真费了点时间.下面是他的结构.

----#page
------#header
------#content
--------#content_body
--------#content .content_sidebar
------#footer

然后我就重点修改了下content_body和.content_sidebar的定位.

#content{
    width: 100%;
}
#content_body{
    width: 75%;
    float: left;
}
.content_sidebar{
    width: 25%;
    float: left;
}

这下在chrome下显示正常了.不过在ff中又崩溃了. 然后纠结了将近5分钟.尝试性的修改

.content_sidebar{
     float:right;
}

这下竟然全都显示正常了. 疑问就来了. 我记得W3C盒子模型.在float上应该没差别啊.为什么float左,右显示不正确呢? 然后我就在ff下写了一个测试页面.









 


 

显示正常…所以我怀疑是因为作者某个地方思考不完善,冗余定位导致.[后来在某一个id标签和类标签混合处发现问题所在…哎. 定位真是个纠结的事情.]
