Subversion Server搭建.[apache2]

Aug 13th, 2011

好久没有更新了. 恩. 各种乱七八糟的事情. 舆情监测的代码好久都没有动了. 开学之后要好好恶补下知识.争取能把这个项目当作毕设来做. 今天搭建了下subversion. 之前是用hg的. 因为舆情监测这个项目, 就转到svn上了.不过没什么太大的差别. 这次的搭建环境是ubuntu.` 恩.Let’s Begin.

aptitude install subversion subversion-tools
aptitude install apache2 libapache2-svn

这个就是基本的准备工作.安装好之后. 就开始配置. 首先,我们先建立版本库.

mkdir -p /var/www/svn
svnadmin create /var/www/svn

建立好版本库之后,我们需要修改一些配置文件.

#vim conf/svnserver.conf
anon-access read
auth-access write
password-db passwd
#vim conf/passwd
xiaomo 123123123

之后,我们便可以启动svn server.

svnserver -d -r /var/www/svn

此时,便可以以经典的svn://方式来使用. 然后,我们开始配置apache2.

#vim /etc/apche2/sites-available/default

        DAV svn
        SVNPath /var/www/svn
        AuthType Basic
        AuthName "SVN Auth"
        AuthUserFile /etc/subversion/passwd

然后重启apache2就可以了. 如果出现不能访问的情况,请注意版本库的权限.

ls -l /var/www | grep svn
#drwxr-xr-x 3 root root 4096 2011-08-13 15:23 svn/

至于authz的配置. 用到再说.` 恩.忘了说怎么导入以及如何设置用户了.. 导入的话.

svn import ~/code svn://127.0.0.1/ --username xiaomo --password 123123

然后设置用户的话,我们用apache的htpasswd

#first time
htpasswd  -c /etc/subversion/passwd xiaomo
#after
htpasswd -m /etc/subversion/passwd xiaomo
